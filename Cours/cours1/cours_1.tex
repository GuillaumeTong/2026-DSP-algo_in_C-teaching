\begin{frame}\frametitle{Qu'est-ce qu'un programme?}
  Wikipedia:
  \begin{quote}
    \textcolor{NavyBlue}{\og Un programme informatique est un ensemble d'opérations
    destinées à être exécutées par un ordinateur.\fg{}}
  \end{quote}
 \begin{itemize}
\item  C'est la description d'une méthode de resolution d'un problème : une suite d'instructions qui traitent les données du problème à résoudre, jusqu'à aboutir à une solution.
\end{itemize}

\vspace{0.5cm}

Il existe plusieurs catégories de programmes :
\begin{itemize}
  \item \textcolor{NavyBlue}{Programme source }(langage de programmation).\\
    Compilable ($\Rightarrow$ binaire) (C,fortran) ou interprétable
    (Java,Python).\medskip
  \item \textcolor{NavyBlue}{Programme binaire }(langage machine).\\
    exécutable par un microprocesseur.



\end{itemize}
\end{frame}
 \begin{frame}
 \frametitle{Exécution d'un programme}
            
\begin{itemize}
\item Les traitements décrits par les instructions sont appliqués aux données (\textcolor{NavyBlue}{\texttt{entrées}}),
  \vspace{0.2cm}
\item Les données ainsi transformées permettent d'aboutir aux solutions recherchées (\textcolor{NavyBlue}{\texttt{sorties}}).
\end{itemize}
\bigskip

Le système d'exploitation\footnote{(windows, macos, ios, linux,  android,...)} \og démarre\fg{} un programme binaire : \\
\textcolor{NavyBlue}{il ordonne au   processeur d'exécuter les instructions du programme.}\\

\vspace{0.2cm}

Par exemple, lorsqu'on double-clique sur une icône, ou bien lorsqu'on tape le nom du ficher exécutable en ligne de commande.
\end{frame}

\begin{frame}\frametitle{$1$ programme, $\infty$ exécutions \emph{différentes}}
  Ne pas confondre:\medskip
  \begin{itemize}
  \item Le \textcolor{NavyBlue}{programmeur} qui \emph{programme} :
    \begin{itemize}
    \item \emph{prévoit} \emph{chaque} cas possibles à l'avance
    \item décrit la (sous-)recette à suivre dans chacun des cas
    \end{itemize}\medskip
  \item Le \textcolor{NavyBlue}{processeur} qui \emph{exécute}
    \begin{itemize}
    \item ne fait que suivre la \og recette\fg{} pas à pas
    \item ne fera pas tous les cas lors d'\emph{une} exécution
    \end{itemize}
  \end{itemize}
\end{frame}

\frame
{
 \frametitle{Les langages de programmation}

 \textbf{\textcolor{NavyBlue}{Les langages de haut niveau :}} (Ex : C, Ada, Pascal, Cobol, Java, OCaml, Python).
 \begin{itemize}
 \item fournissent des constructions sophistiquées qui facilitent l'écriture des programmes.
 \item sont comprehensibles par les humains, mais pas directement exécutables par les machines.
 \item $\Rightarrow$ Traduction en langage machine avant son exécution.
  \end{itemize}

 \vspace{0.5cm}
 
 \textbf{\textcolor{NavyBlue}{Les langages de bas niveau :}} (langages cibles ou natifs)
\begin{itemize}
\item instructions propres à chaque machine (SPARC/Sun, Intel/PC, etc).
\item codés en binaire et directement exécutables par chaque machine.
\end{itemize}


}

\frame
{
 \frametitle{Description d'un langage de haut niveau}

 \textbf{\textcolor{NavyBlue}{Les types des données :}} utilisés pour modéliser les données. \\

\textcolor{NavyBlue}{Exemple :} Le type \texttt{\textcolor{NavyBlue}{int}} en \texttt{C} sert à modéliser les nombres entiers.\\
\vspace{0.4cm}

\textbf{\textcolor{NavyBlue}{La syntaxe : }} règles de formation textuelle des instructions.\\
\textcolor{NavyBlue}{ Exemple :} pour écrire l'expression mathématique $\textcolor{NavyBlue}{1 \leq x \leq 7}$, une syntaxe possible en \texttt{C} est : $\texttt{\textcolor{NavyBlue}{1 <= x \&\& x <= 7}}$.\\
\vspace{0.4cm}
 \textbf{\textcolor{NavyBlue}{La sémantique :}} règles qui précisent "le sens" des contructions syntaxiques à l'exécution et  la cohérence entre types. \\
\textcolor{NavyBlue}{Exemples : }
\begin{itemize}
  \item $\texttt{\textcolor{NavyBlue}{4+3*2}}$ équivaut à la valeur entière $\texttt{\textcolor{NavyBlue}{10}}$
\item \texttt{\textcolor{NavyBlue}{"bonjour"$\texttt{*2}$}} est correct syntaxiquement, mais non sémantiquement : $ \texttt{\textcolor{NavyBlue}{*}}$ ne peut pas s'appliquer à une chaîne de caractères
\end{itemize}


}

\frame
{
 \frametitle{Compilation vs Interprétation}
 \vspace{0.2cm}
Il faut  \textbf{\textcolor{NavyBlue}{traduire}} le langage de haut niveau vers le langage machine spécifique à un processeur $\Rightarrow$ il peut lire les instructions et les exécuter. \\
 \vspace{0.2cm}
\textbf{\textcolor{NavyBlue}{2 méthodes :}}
\begin{itemize}
\item \texttt{\textcolor{NavyBlue}{Compilation : }} traduction de toutes les instructions, puis exécution de la traduction : le résultat dépend de l'ordinateur visé \\
\item \texttt{\textcolor{NavyBlue}{Interprétation :}} conversion et exécution de chaque instruction les unes derrière les autres : plus fexible mais plus lent.
\end{itemize}

\begin{frame}[fragile]\frametitle{Comment mémoriser les opérations effectués ?}
Considérons les fonctions :
\begin{itemize}
  \item \tikz \coordinate (lireInt);\lstinlineCv!lireInt();! qui lit un entier tapé au clavier, on dit qu'elle \textcolor{NavyBlue}{retourne} l'entier tapé. \\
  \vspace{0.2cm}
\item  \tikz \coordinate (lireInt);\lstinlineCv!ecrireInt(int a);! qui affiche à l'écran l'entier \textcolor{NavyBlue}{\texttt{a}} passé en paramètre.

\end{itemize}

\vspace{0.4cm}
  On souhaite lire un entier $\textcolor{NavyBlue}{\texttt{n}}$ au clavier, puis afficher la
    valeur de \textcolor{NavyBlue}{\texttt{$n + n^2$}}. \\


\begin{itemize}
  \item \lstinlineCv!ecrireInt(lireInt()+lireInt())! fait 2 lectures clavier.
  \item comment faire pour utiliser plusieurs fois le résultat du
    premier \lstinlineCv!lireInt()!?
   \end{itemize}

\vspace{0.4cm}
  $\Rightarrow$ Stockage dans une \textbf{\textcolor{NavyBlue}{\textit{variable}}} de la valeur retournée par (du résultat)  \lstinlineCv!lireInt()! pour l'utiliser plus tard.
 
\end{frame}


\begin{frame}[fragile]\frametitle{Qu'est ce qu'une variable?}
Une \textbf{\textcolor{NavyBlue}{\textit{variable}}} est une portion de mémoire qui a un nom (en C: n'importe quel nom commençant par une lettre). \\
\vspace{0.3cm}
Elle a les propriétés suivantes :

\vspace{0.1cm}
\begin{itemize}
  \item elle doit être suffisamment grande pour contenir la donnée \\(ex: contenir un \lstinlineCv!int!)
  \vspace{0.1cm}
  \item elle change de valeur au cours de l'exécution du programme
\end{itemize}
 
\vspace{0.3cm}

Il est nécessaire de la \emph{\textcolor{NavyBlue}{déclarer}} avant de l'utiliser. \\
\vspace{0.3cm}

Les opérations possibles sur les variables sont :

\vspace{0.1cm}
    \begin{itemize}
    \item \emph{\textcolor{NavyBlue}{l'affectation}}: remplacer sa valeur courante par une autre valeur.
    \vspace{0.1cm}
    \item \emph{\textcolor{NavyBlue}{l'utilisation}}: utiliser sa valeur courante dans une instruction.
 
  \end{itemize}
\end{frame}


\begin{frame}[fragile]\frametitle{Les variable : exemple en \texttt{C}}
\vspace{0.2cm}

\begin{itemize}
  \item \emph{\textcolor{NavyBlue}{Déclaration}}: 
\begin{lstlistingC}[]
  /*@\tikz \coordinate(typ);@*/int /*@\tikz\coordinate(nme);@*/n ;
  int x,m ;
\end{lstlistingC}
    % \tikz \coordinate (typ);\lstinlineCv!int! \tikz \coordinate (nme);\lstinlineCv!n;!
  \item \emph{\textcolor{NavyBlue}{Affectation}}:
\begin{lstlistingC}[]
  n = 12 ;
  x = 3+4 ;
  m = lireInt()+7 ;
\end{lstlistingC}
    
    \begin{itemize}
    \item signe \og égal\fg{} (\lstinlineCv!=!) ne signifie pas
      égalité (voir\lstinlineC!==!)
    \item \lstinlineCv!x = 12;! signifie \og mettre la valeur 12
      dans la variable \texttt{\textcolor{NavyBlue}{x}}\fg.
    \end{itemize}\medskip
  \item \emph{\textcolor{NavyBlue}{Utilisation}}:
\begin{lstlistingC}[]
    ecrireInt(n+8+g(n+3));
    f(n + n*n + x * m);
    x = g(n +3 , m - 7);
\end{lstlistingC}

  \end{itemize}

\frame
{
 \frametitle{Notion de type de données}
Toutes les variables ont un \textbf{\textcolor{NavyBlue}{type}}, il définit :
\vspace{0.2cm} 
\begin{itemize}
\item la taille de l'espace mémoire réservé pour celle-ci.
\vspace{0.2cm} 
\item l'ensemble des valeurs que peut prendre un objet informatique ainsi que les opérations permises sur ces valeurs.
\end{itemize}
\vspace{0.5cm}
  \textbf{\textcolor{NavyBlue}{Un objet informatique}} est toute entité à laquelle un type est associé (variable, constante, fonction).\\
  
\vspace{0.7cm}
\textbf{\textcolor{NavyBlue}{Exemple : le type entier \texttt{int} :}}
\begin{itemize} 
\item Ensemble de valeurs : $\mathbb{N}$ \\
\item Ensemble d'opérations sur ces valeurs $(+,-,*,/,==, \ldots)$
\end{itemize}

}
\frame
{
 \frametitle{Intérêt du concept de type}
Fournir à la machine les \textbf{\textcolor{NavyBlue}{informations}} nécessaires pour qu'elle soit en mesure de \textbf{\textcolor{NavyBlue}{vérifier que l'utilisation}} des objets d'un programme est bien \textbf{\textcolor{NavyBlue}{conforme}} à l'intention préalable du programmeur :
\vspace{0.1cm}
\begin{itemize}
\item  Lorsque nous déclarons un \textbf{\textcolor{NavyBlue}{objet (variable, constante, fonction, ...)}}, nous spécifions les contraintes de son utilisation, l'ensemble des valeurs qui pourront lui être associées et ainsi  le \textbf{\textcolor{NavyBlue}{cadre légitime de son utilisation}}.
\vspace{0.1cm}
\item L'utilisation de cet objet sera précédée d'une \textbf{\textcolor{NavyBlue}{vérification effectuée par le compilateur}}. La valeur qu'il prend appartient-elle à cet ensemble ? Son utilisation est-elle conforme aux contraintes spécifiées ?
\end{itemize}
\vspace{0.3cm}
\textbf{\textcolor{NavyBlue}{Le concept de type permet d'établir un lien sémantique entre la déclaration et l'utilisation d'un objet}}.

}

\frame
{
 \frametitle{Opérateurs et expressions}
\begin{itemize}
\item Un \textbf{\textcolor{NavyBlue}{opérateur}} permet de réaliser des calculs sur une ou plusieurs valeurs appelées \textbf{\textcolor{NavyBlue}{opérandes}}.
\vspace{0.2cm}
\item Il existe des opérateurs unaires (une seule opérande), binaires et ternaires (ex: \texttt{x++}, $\mathtt{9*x}$)
\vspace{0.2cm}
\item Les \textbf{\textcolor{NavyBlue}{expressions}} sont des constructions décrivant des opérations à l'aide d'opérateurs et d'opérandes
\vspace{0.2cm}
\item Les opérandes d'une expression peuvent être des constantes, des variables valuées ou des appels de fonction :
\begin{itemize}
\item \texttt{(34 + min(x,y))} où \texttt{x} et \texttt{y} sont les paramètres de la fonction \texttt{min}.
\item $\mathtt{2>5}$ est une expression booléenne dont le résultat est faux
\item $\mathtt{!(2>5)}$ est une expression booléenne lue : \texttt{non 2 plus grand que 5}
 \end{itemize}
\vspace{0.2cm}
\item \textcolor{NavyBlue}{Une expression correspond toujours à la valeur qu'elle calcule}
\end{itemize}
}
\frame
{
 \frametitle{Opérateurs arithmétiques}
\begin{itemize}
\item \textbf{\textcolor{NavyBlue}{Opérateurs sur les types entiers}} (\texttt{byte, short, int, long})\\
\begin{itemize}
\item \texttt{+, -, *, /, \%}
\vspace{0.2cm}
\item Exemples : \texttt{9/4 = 2, 9\%4 = 1}
\end{itemize}
\vspace{0.5cm}
\item \textbf{\textcolor{NavyBlue}{Opérateurs sur les types réels}} (\texttt{double, float})\
\begin{itemize}
\item \texttt{+,-,*,/}
\vspace{0.2cm}
\item Exemple : \texttt{9/4 = 2,25}
\end{itemize}
\vspace{0.5cm}
\item \textbf{\textcolor{NavyBlue}{Opérateurs relationnels}}
\begin{itemize}
\item \texttt{<, >, <=, >=, ==, !=}
\vspace{0.2cm}
\item Exemples : \texttt{5 > x, a == 'b', (a != 0) \&\& (a <= 100)}
\end{itemize}
\end{itemize}
}
\frame
{
 \frametitle{Précédence des opérateurs (1/2)}
Les expressions composées de plusieurs opérateurs sont évaluées de la \textbf{\textcolor{NavyBlue}{gauche vers la droite}}, selon des \textbf{\textcolor{NavyBlue}{règles de précédence}} indiquant la priorité des opérateurs les uns par rapport aux autres.
\vspace{1cm}
\begin{itemize}
\item \texttt{2 + 3 * 4} $\Rightarrow$  équivaut à \texttt{2 +(3*4)} \\
$\Rightarrow$ s'évalue en \texttt{14} et non pas en \texttt{20}
\vspace{0.5cm}
\item \texttt{2 + 3 > 4} $\Rightarrow$ s'évalue en \texttt{vrai}
\vspace{0.5cm}
\item \texttt{2 + (3 > 7)} $\Rightarrow$ produit une erreur
\end{itemize}
}

\texttt{+ -}\\
\texttt{< <= > >=} \\
\texttt{== !=}\\
\texttt{\&\&}\\
\texttt{||} \\
 \end{tabular}
\end{center}
}

\begin{frame}[fragile]\frametitle{Qu'est ce qu'une instruction ?}
  \begin{center}
    \lstinlineCv!ecrireString("Hello world\n");!
  \end{center}\vfill

  Une \emph{\textcolor{NavyBlue}{instruction}}
  \begin{itemize}
  \item morceau de programme/recette = morceau de texte,
  \item se termine (en C, C++, \texttt{C}) par un point-virgule: \lstinlineCv!;!
  \item opération \emph{réalisée} à l'exécution,
  \end{itemize}\vfill

  \begin{minipage}[t]{.5\linewidth}
    Exemples:
    \begin{itemize}
    \item \og battez 2 oeufs entiers\fg{}
    \item \lstinlineCv!ecrireString("Toto\n");!
    \item \lstinlineCv!x = y + 3;!
    \item \lstinlineCv!return (3);!
    \end{itemize}
  \end{minipage}\hfill
  \begin{minipage}[t]{.3\linewidth}
    Contre-exemples:
    \begin{itemize}
    \item \og 2 oeufs\fg{}
    \item \lstinlineCv!"Toto\n"!
    \item \lstinlineCv!y+3!
    \item \lstinlineCv!(3)!
    \end{itemize}
  \end{minipage}
\end{frame}


\begin{frame}[fragile]\frametitle{Qu'est ce qu'une expression ?}
  \begin{center}
    \lstinlineCv!"Hello world\n"!\hfill

  Une \emph{\textcolor{NavyBlue}{expression}}
  \begin{itemize}
  \item morceau de programme/recette = morceau de texte,
  \item une \emph{partie} d'une instruction
  \item qui correspondra à une \emph{\textcolor{NavyBlue}{valeur}} à l'exécution,
  \end{itemize}\vfill

  \begin{minipage}[t]{.3\linewidth}
    Contre-exemples:
    \begin{itemize}
    \item \og 2 oeufs\fg{}
    \item \lstinlineCv!"Toto\n"!
    \item \lstinlineCv!y+3!
    \item \lstinlineCv!(3)!
    \end{itemize}
  \end{minipage}\hfill
  \begin{minipage}[t]{.5\linewidth}
    Exemples:
    \begin{itemize}
    \item \og battez 2 oeufs entiers\fg{}
    \item \lstinlineCv!ecrireString("Toto\n");!
    \item \lstinlineCv!x = y + 3;!
    \item \lstinlineCv!return (3);!
    \end{itemize}
  \end{minipage}
\end{frame}

\begin{frame}[fragile]\frametitle{Instruction VS expression}
   Quelques règles :
   \medskip
\begin{center}
    \begin{tabular}{l|l}
      Instruction & Expression \\\hline
      se termine par \lstinlineCv!;!& fait partie d'une instruction\\[.5em]
      aura un \tikz \coordinate (effet);\emph{effet} à l'exécution & aura une \tikz \coordinate (valeur);\emph{valeur} à l'exécution\\[.5em]
      n'a pas de type (\lstinlineCv!void!) & a un type (\lstinlineCv!int!,\lstinlineCv!double!,etc)
    \end{tabular}\vfill
\end{center}
En \texttt{C}:
    \begin{itemize}
    \item les expressions sont parfois aussi des instructions.
    \item retournent une valeurs ET ont un effet.
    \item ex: \lstinlineCv!x = 3!
      \begin{itemize}
      \item effet: met la valeur 3 dans la variable \lstinlineCv!x!
      \item valeur: 3.
      \item code bizarre: x = (y = f(x,y)) \emph{à éviter}
      \end{itemize}
    \end{itemize}

\begin{frame}\frametitle{Les sous-programmes}

\textcolor{NavyBlue}{Un sous-programme} est un \og{}bout de programme\fg{}
\begin{itemize}
   \item qui utilise des \emph{\textcolor{NavyBlue}{paramètres}} (ou arguments)
  \item qu'on peut appeler autant qu'on veut \emph{\textcolor{NavyBlue}{avec des paramètres différents}}
  \item qui peut avoir des \emph{\textcolor{NavyBlue}{effets}} (modif. écran ou mémoire, etc)
  \item qui peut avoir {\color{red}un} \emph{\textcolor{NavyBlue}{résultat}} (valeur de
    \og{}retour\fg)
  \end{itemize}\hfill

\vspace{0.4cm}
Il peut être :

  \begin{overprint}
    \onslide<2->{
      \begin{itemize}
      \item écrit par soi-même
      \item ou par un autre programmeur
      \item code source connu/inconnu
              \end{itemize}
    }
  \end{overprint}
\end{frame}
