\frame
{
 \frametitle{Notion de sous-programme}

\begin{itemize}
\item {\bf \textcolor{NavyBlue}{Problème :}} Lorsqu'un programme est long on ne peut le coder en entier dans la fonction principale.
\vspace{0.4cm} 
\item {\bf \textcolor{NavyBlue}{Idée :}} Décomposer le programme en plusieurs sous-programmes plus petits.
\vspace{0.4cm} 
\item  On utilise pour cela les \textcolor{NavyBlue}{sous-programmes (procédures ou fonctions)}.

\end{itemize}
}

\frame
{
 \frametitle{Notion de sous-programme}

\begin{itemize}

\item  Un sous-programme possède un nom, des variables, des instructions, un début et une fin. 
\vspace{0.4cm}
\item l'exécution d'un sous-programme est invoquée directement dans la fonction \texttt{\textcolor{NavyBlue}{main}}, ou dans un sous-programme invoqué dans le fonction  \texttt{\textcolor{NavyBlue}{main}}. 
\vspace{0.4cm}
\item {\bf \textcolor{NavyBlue}{Définition :}}  Un \textcolor{NavyBlue}{sous-programme} est un élément de programme nommé et éventuellement paramétré, que l'on définit afin de pouvoir ensuite l'appeler par son nom en affectant, s'il y a lieu, des valeurs aux paramètres.
\end{itemize}
}


\frame
{
 \frametitle{Les sous-programmes}


\begin{itemize}
\item {\bf \textcolor{NavyBlue}{Les intérêts :}}
\vspace{0.2cm}
\begin{itemize}
\item le gain de place en mémoire pour le code du programme : lorsqu'un sous-programme est appelé plusieurs fois, dans une boucle,
\vspace{0.2cm}
\item les notions d'abstraction et de modularité : utiliser un appel de sous-programme permet de d'écrire une application en faisant abstraction (en dissimulant) les détails de la fonction que réalise ce sous-programme.
\end{itemize}
\vspace{0.5cm}
\item {\bf \textcolor{NavyBlue}{Quelques règles :}}
\vspace{0.2cm}
\begin{itemize}
\item Un sous-programme doit être homogène : il doit réaliser une tâche précise, formant un tout.
\item Il doit être de taille "raisonnable". La compréhension et la gestion du programme en dépend.
\end{itemize}
\end{itemize}
}


\begin{frame}[fragile]

 \frametitle{Les sous-programmes}


\begin{itemize}
\item La définition du sous-programme est composée d'une spécification qui indique son nom, ses paramètres (ou arguments) avec leurs caractéristiques (nom, type) et d'un
corps comprenant éventuellement des déclarations d'objets locaux au sous-programme et les instructions à exécuter.
\vspace{0.2cm}
\item Déclaration d'un sous-programme en \texttt{C}:\\
\vspace{0.3cm}

\texttt{typeRetour nomFonction (typeAr1 arg1, typeAr2 arg2, ...)}\\
\vspace{0.2cm}

\textcolor{NavyBlue}{Exemple en C :}\\

\begin{lstlisting}
int perimetre(int largeur, int longueur)
\end{lstlisting}

\end{itemize}
\end{frame}


\frame
{
 
\begin{center} {\huge \textbf{\textcolor{NavyBlue}{Les types de sous-programmes}}} \end{center}
}


\frame
{
 \frametitle{Types de sous-programmes}

Il existe deux types de sous-programmes :
\vspace{0.2cm}
\begin{itemize}
\item {\bf \textcolor{NavyBlue}{Les procédures :}} sous-programme nommé, ne renvoyant pas de résultat, et modifiant généralement son environnement. 

\vspace{0.2cm}
\item {\bf \textcolor{NavyBlue}{Les fonctions :}} sous-programme nommé, renvoyant une valeur.


\end{itemize}



}




\begin{frame}[fragile]
 \frametitle{Les procédures}


 {\bf \textcolor{NavyBlue}{Les procédures :}} sous-programme nommé, ne renvoyant pas de résultat, et modifiant généralement son environnement. \\

\vspace{0.2cm}

 {\bf \textcolor{NavyBlue}{Exemple :}}
 
\begin{small}
\begin{lstlisting}
#include "inout.h"

void afficher(int i) {
  EcrireInt(i);
} 

void main()
{ 
  afficher(3);
}
\end{lstlisting}
\end{small}
\vspace{0.2cm}

\begin{center}
{\bf \textcolor{NavyBlue}{L'appel d'une procédure constitue une instruction en lui-même}}
\end{center}

\end{frame}


\begin{frame}[fragile]
 \frametitle{Les fonctions}


 {\bf \textcolor{NavyBlue}{Les fonctions :}} sous-programme nommé, renvoyant une valeur.\\

\vspace{0.2cm}

 {\bf \textcolor{NavyBlue}{Exemple :}}
 
\begin{small}
\begin{lstlisting}
int carre(int n) {
  return n*n;
} 

void main()
{ 
  int n;
  n = carre(2);
  afficher(n);
}
\end{lstlisting}
\end{small}
\vspace{0.2cm}



\end{frame}



\begin{frame}[fragile]
 \frametitle{Les fonctions}



Les fonctions exécutent des instructions et en plus retournent une valeur. \\

\vspace{0.5cm}
L'appel d'une fonction est remplacé dans le programme appelant par la valeur renvoyée.

\vspace{0.5cm}

\begin{center}
{\bf \textcolor{NavyBlue}{L'appel d'une fonction représente une valeur qui doit être utilisée à l'intérieur d'une instruction.}}
\end{center}

\end{frame}

\begin{frame}[fragile]

 \frametitle{Communication entre programme appellant et sous-programmes}
Les données et les résultats d'un sous-programme proviennent d'une communication entre le programme appelant et le sous-programme. \\
\vspace{0.3cm}

3 modes de communication sont possibles : \\
\begin{itemize}
\item l'utilisation de variables communes à l'appelant et à l'appelé : \\
ce sont les {\bf \textcolor{NavyBlue}{variables globales}}.
\vspace{0.2cm}
\item {\bf \textcolor{NavyBlue}{le passage de paramètres}}, pour les procédures et les fonctions
\vspace{0.2cm}
\item {\bf \textcolor{NavyBlue}{la valeur de retour}} pour les fonctions
\end{itemize}
\end{frame}




\begin{frame}[fragile]

 \frametitle{Les variables locales ou globales}
L'endroit où est déclaré une variable est très important car il détermine dans quel(s) sous-
programme(s) elle va pouvoir être utilisée. \\
\vspace{0.4cm}
\begin{itemize}
\item {\bf \textcolor{NavyBlue}{Une variable locale}} est déclaré dans un sous-programme et n'est utilisable que dans le
sous-programme où elle a été déclarée.
\vspace{0.4cm}

\item {\bf \textcolor{NavyBlue}{Une variable globale }} est déclarée à l'extérieur du programme principal et des sous-programmes : elle
 est commune à l'ensemble des sous-programmes et du programme principal et est utilisable partout.
\end{itemize}


\end{frame}

\begin{frame}[fragile]

 \frametitle{Appel d'un sous-programme et passage de paramètre} 
L'appel d'un sous-programme se fait en mentionnant son nom, suivi des
paramètres effectifs figurant entre parenthèses et séparés par des virgules.\\
\vspace{0.5cm}

Il existe deux mécanismes importants de substitution entre paramètres
effectifs et paramètres formels :\\
\begin{itemize}
\item le passage \textbf{\textcolor{NavyBlue}{par valeurs,}}
\item le passage \textbf{\textcolor{NavyBlue}{par référence.}}
\end{itemize}
\end{frame}


\begin{frame}[fragile]

 \frametitle{Le passage de paramètre par valeur}
La valeur du paramètre effectif est recopiée dans le paramètre formel à
l'entrée du sous-programme. Le sous-programme travaille sur une copie du
paramètre effectif. Celui-ci n'est pas modifié à l'issue du sous-programme. 

\end{frame}

Le résultat de l'exécution du programme est l'affichage suivant : \\

 \texttt{\textcolor{NavyBlue}{avant échange, a = 3, b= 5}}\\
 \texttt{\textcolor{NavyBlue}{après échange, a = 3, b= 5}}\\
\vspace{0.7cm}

 L'appel à la procédure \texttt{\textcolor{NavyBlue}{echanger(a,b)}} est un appel par valeur, les paramètres formels  \texttt{\textcolor{NavyBlue}{x}} et  \texttt{\textcolor{NavyBlue}{y}} sont remplacés par  \texttt{\textcolor{NavyBlue}{a=3}} et  \texttt{\textcolor{NavyBlue}{b=5}}, elle reçoit des copies des paramètres \texttt{\textcolor{NavyBlue}{a}} et \texttt{\textcolor{NavyBlue}{b}}.\\

\begin{frame}[fragile]

 \frametitle{Le passage de paramètre par référence}


 \textbf{\textcolor{NavyBlue}{Définition :}} une variable de type référence est une variable dans laquelle est stockée une adresse. \\
\vspace{0.7cm}

L'adresse du paramètre effectif est communiquée au sous-programme qui travaille alors directement sur la variable passée en paramètre et non sur une copie locale. Le
sous-programme peut alors modifier la valeur d'une variable du programme qui l'a appelé.\\
\vspace{0.7cm}
Lorsqu'un tableau est passé en paramètre, c'est sa référence qui est en fait passée (i.e. son adresse), le tableau d'origine et celui vu par la procédure occupent donc le même espace mémoire.
\end{frame}


\begin{frame}[fragile]

 \frametitle{Le passage de paramètre par référence : Exemple}

La procédure \texttt{\textcolor{NavyBlue}{echangerTab}} échange les élements des d'indices \texttt{\textcolor{NavyBlue}{i}} et \texttt{\textcolor{NavyBlue}{j}} du tableau \texttt{\textcolor{NavyBlue}{t}} qui lui sont passé en paramètre.
 
\begin{lstlisting}
echangerTab(int t[], int i, int j) {
  int temp;
  temp = t[i];
  t[i]=t[j];
  t[j]=temp;
}
\end{lstlisting}
La fonction principale \texttt{\textcolor{NavyBlue}{main}} appelle la procédure \texttt{\textcolor{NavyBlue}{echangerTab}} sur le tableau \texttt{\textcolor{NavyBlue}{t}} et les indices \texttt{\textcolor{NavyBlue}{1}} et \texttt{\textcolor{NavyBlue}{3}}.
\begin{lstlisting}
void main( ) {
  int t[]= {3, 4, 24};
  ecrireString("avant tab=");
  ecrireInt(t[0]); ecrireInt(t[1]); ecrireInt(t[2]);
  echangerTab(tab,1,2);
  ecrireString("avant tab=");
  ecrireInt(t[0]); ecrireInt(t[1]); ecrireInt(t[2]);
}
\end{lstlisting}


\end{frame}

\begin{frame}[fragile]

 \frametitle{Le passage de paramètre par reférence : Exemple}
Le résultat de l'exécution du programme est l'affichage suivant : \\

 \texttt{\textcolor{NavyBlue}{avant tab=3,4,24}}\\
 \texttt{\textcolor{NavyBlue}{apres tab=3,24,4}}\\
\vspace{0.2cm}

 L'appel à la procédure \texttt{\textcolor{NavyBlue}{echangerTab(tab,1,2)}} est un appel par valeur, mais la valeur de tab est sa référence, les paramètres formels \texttt{\textcolor{NavyBlue}{t}},  \texttt{\textcolor{NavyBlue}{i}} et  \texttt{\textcolor{NavyBlue}{j}} sont remplacés par \texttt{\textcolor{NavyBlue}{ref t}},  \texttt{\textcolor{NavyBlue}{i=1}} et  \texttt{\textcolor{NavyBlue}{j=4}}, elle reçoit des copies de ces paramètres.\\

\vspace{0.3cm}
\end{frame}
