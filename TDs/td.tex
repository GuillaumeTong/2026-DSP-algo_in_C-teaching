\noindent {\large \textbf{1. Conception de programmes}}\\

\vspace{0.2cm}

\noindent Pour écrire un programme il faut suivre les étapes suivantes :
\begin{enumerate}
\item Déterminer les Entrées du programme
\item Déterminer les sorties du programme
\item Concevoir l'algorithme qui à partir des données produira la sortie
\item Traduire l'algorithme sous forme d'une fonction \texttt{C}
\end{enumerate}

Pour chacuns des problèmes suivants détaillez les étapes de conception décrites ci-dessus.

Quelle est la valeur des expressions suivantes ?
\begin{enumerate}

    nf = ne
\end{verbatim}

\end{correction}
\end{exo}

\begin{exo}[Vrai ou faux]~
\begin{enumerate}
\item Tous les éléments d'un tableau ont le même type
\item La taille du tableau peut être fixée au moment de sa déclaration
\item La taille du tableau peut être fixée au moment de sa création
\item Les instructions suivantes sont-elles valides ? Si non corrigez.
  \begin{itemize}
\item \texttt{int i = \{1,2,3\};}
\item \texttt{double[]  d;}
\item \texttt{float f[3] ;}
\item \texttt{char r[] = = \{1,2,3\};}
\item \texttt{int i[] = (3,4,6,8);}
\item \texttt{float f[] = \{3.2,4.5,5.6\};}
\item \texttt{char [] c = \{'a', 'b','c'\};}
\end{itemize}
  \end{enumerate}
\begin{correction}
  \begin{enumerate}
    \item Tous les éléments d'un tableau ont le même type : \textbf{vrai}
\item La taille du tableau peut être fixée au moment de sa déclaration : \textbf{faux}
\item La taille du tableau peut être fixée au moment de sa création : \textbf{vrai}
\item Les instructions suivantes sont-elles valides ? Si non corrigez.
  \begin{itemize}
  \item \texttt{int i = \{1,2,3\};}\textbf{non}\\
\texttt{int i[] = \{1,2,3\};} ou \texttt{int i[3] = \{1,2,3\};}
\item \texttt{double[]  d;}\textbf{non}\\
  \texttt{double d[]  ;}
\item \texttt{float f[3] ;}\textbf{oui}
\item \texttt{char r[] = \{1,2,3\};}\textbf{non}\\
  \texttt{char r[] =  \{'a', 'b','c'\};}
\item \texttt{int i[] = (3,4,6,8);}\textbf{non}\\
  \texttt{int i[] = \{3,4,6,8\};} ou \texttt{int i[4] = \{3,4,6,8\};}
\item \texttt{float f[] == \{3.2,4.5,5.6\};}\textbf{non}\\
  \texttt{float f[] = \{3.2,4.5,5.6\};} ou \texttt{float f[3] = \{3.2,4.5,5.6\};}
\item \texttt{char c  [] = \{'a', 'b','c'\};}\textbf{oui}
\end{itemize}
\end{enumerate}

\end{correction}
\end{exo}

Un institut de sondage veut faire une enquête sur les intentions de vote à un référendum. Il y a trois intentions possibles :

\begin{center}

 & oui& non& abstention \\\hline

femme & 9& 47&  13\\\hline
\end{tabular}

La représentation en Java d'un tel tableau se fera de la façon suivante :
\begin{center}
\begin{tabular}{c|c|c|c|}

& 0 & 1 & 2\\\hline
0 &  12& 37& 25\\\hline
1 & 9& 47& 13
\end{tabular}
\end{center}
Ici pour les colonnes, l'indice 0 signifie une intention de vote oui, l'indice 1 signifie non et l'indice 2 signifie l'abstention, et pour les lignes l'indice 0 signifie les hommes et l'indice 1 signifie les femmes.
